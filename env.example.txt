# ========================================
# MATS - 多交易所 API 配置範本
# 檔案名稱：env.example.txt
# ========================================
# 
# 使用說明：
# 1. 複製此檔案並改名為 .env
#    Windows: copy env.example.txt .env
#    Linux:   cp env.example.txt .env
# 2. 填入您使用的交易所 API 金鑰
# 3. 不使用的交易所可以留空或刪除
# 4. 在 exchange_config.js 中選擇要使用的交易所
#
# 注意：改名後的 .env 檔案不要上傳到 Git！
#
# ========================================

# ========== Binance（幣安）==========
# 取得方式：https://www.binance.com/zh-TW/my/settings/api-management
# 需要權限：讀取 + 交易（不要開啟提現）
BINANCE_API_KEY=your_binance_api_key_here
BINANCE_SECRET=your_binance_secret_here

# ========== Bybit ==========
# 取得方式：https://www.bybit.com/app/user/api-management
# 需要權限：讀取 + 交易（不要開啟提現）
BYBIT_API_KEY=your_bybit_api_key_here
BYBIT_SECRET=your_bybit_secret_here

# ========== OKX（歐易）==========
# 取得方式：https://www.okx.com/account/my-api
# 需要權限：讀取 + 交易（不要開啟提現）
# 注意：OKX 需要額外的 Passphrase（創建 API 時自己設定的密碼）
OKX_API_KEY=your_okx_api_key_here
OKX_SECRET=your_okx_secret_here
OKX_PASSWORD=your_okx_passphrase_here

# ========== Bitget ==========
# 取得方式：https://www.bitget.com/zh-TW/account/newapi
# 需要權限：讀取 + 交易（不要開啟提現）
# 注意：Bitget 需要額外的 Passphrase（創建 API 時自己設定的密碼）
BITGET_API_KEY=your_bitget_api_key_here
BITGET_SECRET=your_bitget_secret_here
BITGET_PASSWORD=your_bitget_passphrase_here

# ========================================
# 常見問題
# ========================================
#
# Q: 我只用 Binance，其他交易所要填嗎？
# A: 不用！只需填入您使用的交易所即可。
#
# Q: OKX 和 Bitget 的 PASSWORD 是什麼？
# A: 這是您在創建 API 時自己設定的 Passphrase（密碼短語），
#    不是您的登入密碼。如果忘記了，需要重新創建 API。
#
# Q: API 權限應該如何設置？
# A: 只開啟「讀取」和「交易」權限，
#    絕對不要開啟「提現」權限！
#
# Q: 建議設置 IP 白名單嗎？
# A: 強烈建議！在交易所 API 設置中添加您伺服器的 IP，
#    可以大幅提高安全性。
#
# ========================================
